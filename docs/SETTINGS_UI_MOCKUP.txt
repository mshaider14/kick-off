USER FLOWS:
───────────
1. Page Load
   → Fetches settings from backend
   → Displays default settings for new merchants
   → Shows account information

2. Edit Settings
   → User changes timezone
   → User toggles tracking options
   → User updates notification preferences
   → All changes update form state immediately

3. Save Settings
   → User clicks "Save Settings"
   → Button shows loading state
   → API request sent to backend
   → Validation performed
   → Success: Green toast "✅ Settings saved successfully!"
   → Error: Red toast "❌ Error: [message]"
   → Settings persisted to database

4. Reload
   → Settings loaded from database
   → Form populated with saved values
   → User sees their previous selections
